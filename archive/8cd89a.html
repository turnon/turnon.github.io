nil?的实现

2016-08-24 ruby
当对一个变量调用nil?时，它背后是如何运作的？其实很简单，一切皆对象，和继承。

ruby中所有东西都继承自Object（除了BasicObject）：

Object.methods.grep(/nil/)
#=> [:nil?]


那么，看看Object的nil?是如何实现的（https://github.com/ruby/ruby/blob/trunk/object.c） ：

static VALUE

rb_false(VALUE obj)
{
    return Qfalse;
}

rb_define_method(rb_mKernel, "nil?", rb_false, 0);


基本上，其意思是，调用nil?，则直接地返回false。演示如下

5.nil?
#=> false
false.nil?
#=> false
"nil".nil?
#=> false


那么nil呢？当你使用nil关键字时，你得到的将是一个NilClass的singleton实例

nil.class
#=> NilClass
nil.class.ancestors
#=> [NilClass, Object, Kernel, BasicObject]

NilClass继承自Object，但重写了nil?方法：

static VALUE
rb_true(VALUE obj)
{
    return Qtrue;
}

rb_define_method(rb_cNilClass, "nil?", rb_true, 0);


如果通过ruby来描述，那将是这样：

class Object
  def nil?
    false
  end
end

class NilClass < Object
  def nil?
    true
  end
end

1.nil? #=> false
nil.nil? #=> true

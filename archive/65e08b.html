proxy pattern

2015-05-07 javascript pattern ruby underscore
代理对象需与实际对象接口一样，令调用者察觉不到调用的是代理还是本题
主要有protection（常用于拦截）、virtual（常用于惰性执行）、cache，另有remote、synchronization、counting、firewal

ruby

使用 virtual proxy 延迟实例化

class VirtualAccountProxy

  def initialize(starting_balance=0)
    @starting_balance=starting_balance
  end

  def deposit(amount)
    return subject .deposit(amount)
  end

  def withdraw(amount)
    return subject .withdraw(amount)
  end

  def balance
    return subject .balance
  end

  def subject
    @subject || (@subject = BankAccount.new(@starting_balance))
  end

end


改进，通过block，使代理类与“实例的new”解耦

class VirtualAccountProxy

  def initialize(&creation_block)
    @creation_block = creation_block
  end

  # Other methods omitted ...

  def subject
    @subject || (@subject = @creation_block.call)
  end

end


通过method_missing，直接调用被代理对象的对应方法

class AccountProtectionProxy

  def initialize(real_account, owner_name)
    @subject = real_account
    @owner_name = owner_name
  end

  def method_missing(name, *args)
    check_access
    @subject.send( name, *args )
  end

  def check_access
    if Etc.getlogin != @owner_name
      raise "Illegal access: #{Etc.getlogin} cannot access account."
    end
  end

end


js virtual

见underscore的after

_.after = function(times, func) {
  return function() {
    if (--times < 1) {
      return func.apply(this, arguments);
    }
  };
};

js cache

见underscore的memoize

_.memoize = function(func, hasher) {
  var memoize = function(key) {
    var cache = memoize.cache;
    var address = '' + (hasher ? hasher.apply(this, arguments) : key);
    if (!_.has(cache, address)) cache[address] = func.apply(this, arguments);
    return cache[address];
  };
  memoize.cache = {};
  return memoize;
};

this的binding rule

2014-06-29 javascript
to do: arrow function中的 this 在定义它的时候已经决定了（执行定义它的作用域中的 this），与如何调用以及在哪里调用它无关，包括 (call, apply, bind) 等操作都无法改变它的 this

this的指向与作用域链无关，只看执行时绑定


注意，即使外层函数是explicit binding，内层仍有可能是implicit binding：

var a = {id: 123};

function show_id(){
  console.log(this.id);
  (function(){
    console.log(this.id);
   })();
}

show_id.apply(a); // 输出123和undefined


解决方法，同样explicit，或提前that

提前that/self：

所谓提前that，就是利用closure来引用“外层”的this，上例可改为：

function show_id(){
  console.log(this.id);
  (function(that){
    console.log(that.id);
   })(this);
}


另一例子：

var a = 10;

var obj = {
    foo: function() {
        var that = this;
        return function() {
            console.log(that.a);
        };
    },
    a: 1
};

obj.foo()(); // 1


eval

对于eval函数，其执行时候似乎没有指定当前对象，但实际上其this并非指向window，因为该函数执行时的作用域是当前作用域，即等同于在该行将里面的代码填进去。下面的例子说明了这个问题：

var name = "window";

var Bob = {
    name: "Bob",
    showName: function(){
        eval("alert(this.name)");
    }
};

Bob.showName();    //Bob
